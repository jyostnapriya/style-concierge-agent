[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through how this works under the hood. 
First, when a user enters a query, it starts at the Security Checkpoint node. Here, we run regex checks to redact PII like email and phone numbers, and scan for prompt injections or policy violations.
(pause)
If everything is clean, the flow proceeds to run_orchestration, which activates our main Orchestrator agent.
The Orchestrator coordinates two specialized sub-agents via AgentTools: the weather_agent and the style_recommender.
The weather_agent communicates with our local MCP Server over Stdio to fetch real-time forecasts.
Then, the style_recommender queries the MCP Server to pull style tips, color palettes, and wardrobe basics.
Once the recommendation is generated, the flow hits the human_review node. This is a Human-in-the-Loop step that pauses the workflow and asks the user for explicit approval.
If approved, it goes to the final_output node. If rejected, it routes back to run_orchestration along with the user's feedback to regenerate the recommendation.
If a security event is triggered at the start, the flow routes directly to security_error and denies access.

[3:00 — SECURITY & MCP]
This secure design ensures that prompt injections, PII, and policy-violating queries are stopped at the perimeter before any sub-agents are executed or logged. 
Additionally, the Model Context Protocol server enables our sub-agents to access local tools for weather forecast, style tips, color palettes, and wardrobe basics dynamically using stdio.
